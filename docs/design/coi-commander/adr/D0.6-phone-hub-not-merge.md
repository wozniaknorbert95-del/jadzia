# D0.6 ADR — Phone-first Commander hub (not-merge)

**Status:** Accepted  
**Date:** 2026-07-18  
**Supersedes relation:** Complements D0.5 (hosting); does not reverse greenfield SPA.

## Context

Dowódca needs one smartphone entry point for FlexGrafik control. Separate products exist:

| Surface | Role |
|---------|------|
| Jadzia COI Commander | Business ops (leads, marketing, analytics, agents) |
| Agent OS Mission Control `:3000` | Code orchestration / HITL diffs |
| VCMS | Governance / repo scan |
| Wizard / Design Agent | Customer + mockups |

Merging these codebases would couple auth, deploy risk, and cognitive domains incorrectly.

## Decision

1. **Keep backends separate** (jadzia-core ≠ agent-os ≠ VCMS ≠ zzpackage).
2. **Phone UX hub = Commander** at `/commander/` (same FastAPI unit as D0.5).
3. **System map** on Home: deep-links outward (Agent OS, VCMS docs/path, Wizard).
4. **Mobile shell:** sticky bottom nav + touch targets ≥44px; Home queue first.

## Canonical URLs (prod)

| Link | URL |
|------|-----|
| Commander hub | `https://api.zzpackage.flexgrafik.nl/commander/` |
| Agent OS Mission Control | `http://localhost:3000` (tunnel/VPN as operated) |
| Wizard | `https://zzpackage.flexgrafik.nl/wizard/` |
| Design Agent health | `https://api.zzpackage.flexgrafik.nl/api/v1/design-agent/health` |
| VCMS | local workspace `Flex-vcms/flex-vcms` (governance; not runtime) |

## Consequences

- One cognitive home for Dowódca without false monolith.
- Agent OS / VCMS remain independent deploy units.
- Demand path (`REV-DEMAND-*`) stays in jadzia; hub does not block revenue APIs.
- Future PWA / magic-link auth is a separate ADR (out of scope here).

## Out of scope

- Merge agent-os-ui into commander-ui
- SSO / magic JWT
- Service worker PWA
- Rewrite Marketing/Analytics content views
- Gate D / Mollie / min199
