# D0.17 — Empty / error / stale contracts

**Status:** Spec  
**Date:** 2026-07-18

## Contract

| Stan | Znaczenie | Copy PL (wzór) |
|------|-----------|----------------|
| empty | Fetch OK, zero items | „Brak danych — spokój.” / kontekstowy |
| error | Fetch failed | „Nie udało się pobrać. Spróbuj ponownie.” + retry |
| stale | Data OK but freshness bad | Badge / strip „Dane nieaktualne” + źródło |

## Surfaces

Marketing queue, Analytics tiles/lists, Agents list, Audit list, Home priorities/queue (już częściowo).

Empty ≠ error ≠ stale — nigdy sam „Brak” bez kontekstu przy error.
