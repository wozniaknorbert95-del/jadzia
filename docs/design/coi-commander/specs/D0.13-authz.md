# D0.13 — Multi-Role Authz (N7)

## JWT claims

`sub`: user identifier  
`role`: `dowodca` | `delegat` | `viewer` (default `dowodca` if omitted)  
`scopes[]`: explicit scope list (optional override)

Generate tokens:

```bash
python scripts/jwt_token.py --role dowodca --sub norbert
python scripts/jwt_token.py --role delegat --sub delegat1
python scripts/jwt_token.py --role viewer --sub viewer1
```

## Settings role map (F3.1)

`commander_roles` in `commander_settings` maps `user_id` → role.  
Resolved in `agent/commander/authz.py:resolve_role()` before JWT `role` claim.

## Server-side scopes

| Role | Scopes |
|------|--------|
| Dowódca | `*` |
| Delegat | `marketing:approve`, `marketing:publish`, `marketing:unpublish`, `queue:act`, `leads:act`, `commander:read` |
| Viewer | `*:read`, `commander:read` |

Backend returns **403** on scope violation. UI hide is not security.

## Block escalate-through-UI

Delegat cannot assign Dowódca role (`settings:roles` scope — Dowódca only).
