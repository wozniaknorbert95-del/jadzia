## Summary

Empty `cli/__init__.py` skeleton removed. New professional CLI implemented from scratch.

### What was built

`cli/main.py` — 280-line CLI with 8 commands:

| Command | Description | Example |
|---------|-------------|---------|
| `health` | Check if Jadzia API is reachable | `jadzia health` |
| `status` | Show current agent session status | `jadzia status` |
| `worker` | Check worker health and queue | `jadzia worker` |
| `costs` | Show current cost summary | `jadzia costs` |
| `sessions` | Show active sessions | `jadzia sessions` |
| `logs` | Show recent log entries (NLP, not JSON) | `jadzia logs` |
| `test` | Run connectivity tests (5 checks) | `jadzia test` |
| `version` | Show CLI and API version | `jadzia version` |

Features:
- `--url` flag on every subcommand (customizable endpoint)
- `-q` quiet mode
- Exit codes: 0 = OK, 1 = degraded, 2 = unreachable
- Color-formatted output
- Graceful error handling (requests timeouts, HTTP errors)
- `requests` library requirement
